# 店櫃業績管理系統

## 系統簡介
這是一個基於 PHP + 檔案系統的店櫃業績管理系統，專為 QRS 服裝店櫃設計。系統不使用資料庫，所有資料儲存為 JSON 檔案。

## 系統功能

### 1. 權限管理
- **店櫃人員**: 登打每日業績 + 查看自己店櫃報表
- **業務人員**: 查看負責店櫃報表 (日/月/年)
- **督導人員**: 查看所有店櫃報表 (日/月/年)
- **管理人員**: 全功能 + 修改資料

### 2. 核心功能
- 業績登打與管理
- 日/月/年報表查詢
- 店櫃與人員管理
- 資料匯出 (Excel/CSV)
- 操作日誌記錄

### 3. 資料結構
```
sales/
├── data/
│   ├── users.json          # 使用者資料
│   ├── stores.json         # 店櫃資料
│   ├── sales_summary.json  # 銷售彙總資料
│   └── logs/               # 日誌檔案
├── config/                 # 設定檔
├── admin/                  # 管理功能
├── store/                  # 店櫃功能
├── sales/                  # 業務功能
├── supervisor/             # 督導功能
└── common/                 # 共用函式
```

## 安裝與設定

### 1. 環境需求
- PHP 7.4+
- 檔案寫入權限
- 支援 JSON 擴展

### 2. 快速開始
1. 將整個 `sales` 資料夾上傳到伺服器
2. 確保 `data/` 目錄有寫入權限
3. 訪問 `index.php` 開始使用

### 3. 預設帳號
- **管理員**: admin / admin123
- **督導**: supervisor1 / super123
- **業務**: sales1 / sales123
- **店櫃**: store1 / store123

## 使用說明

### 1. 首次使用
1. 訪問 `test_system.php` 檢查系統狀態
2. 訪問 `import_daily_sales.php` 匯入測試資料
3. 使用預設帳號登入系統

### 2. 業績登打
1. 店櫃人員登入後，點選「登打業績」
2. 選擇店櫃、日期，輸入銷售金額
3. 可選填交易筆數和備註

### 3. 報表查詢
1. 業務/督導登入後，點選「業績報表」
2. 選擇日期範圍和報表類型
3. 可匯出為 CSV 格式

### 4. 資料管理
1. 管理員登入後，可管理店櫃和使用者
2. 可匯入 Excel 格式的業績資料
3. 可查看操作日誌

## 資料匯入

### 1. Excel 格式
系統支援從 Excel 匯入業績資料，格式要求：
- 第一列為店櫃代號
- 第一行為日期（或自動從起始日期計算）
- 資料為數值格式

### 2. 匯入步驟
1. 準備 Excel 檔案
2. 訪問 `import_daily_sales.php`
3. 設定起始日期
4. 執行匯入

## 安全性

### 1. 密碼保護
- 使用 `password_hash()` 加密密碼
- 支援 bcrypt 演算法
- 自動產生隨機密碼

### 2. 權限控制
- 基於角色的存取控制
- Session 超時保護
- 操作日誌記錄

### 3. 輸入驗證
- 所有輸入過濾和驗證
- 防止 SQL 注入（雖然不用資料庫）
- 檔案路徑安全檢查

## 維護與備份

### 1. 定期備份
建議定期備份 `data/` 目錄下的 JSON 檔案：
- `users.json` - 使用者資料
- `stores.json` - 店櫃資料
- `sales_summary.json` - 銷售資料

### 2. 日誌管理
- `logs/audit.log` - 操作日誌
- `logs/error.log` - 錯誤日誌
建議定期清理舊日誌。

### 3. 效能優化
- 資料量過大時，可考慮分割檔案
- 定期清理無效資料
- 啟用 PHP 快取

## 故障排除

### 1. 常見問題
1. **無法寫入檔案**: 檢查目錄權限
2. **JSON 解析錯誤**: 檢查檔案格式
3. **登入失敗**: 檢查密碼雜湊

### 2. 測試工具
- `test_system.php` - 系統測試頁面
- 檢查 PHP 錯誤日誌
- 查看系統日誌檔案

### 3. 聯絡支援
如有問題，請檢查：
1. PHP 版本是否符合要求
2. 檔案權限設定
3. JSON 擴展是否啟用
